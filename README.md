---

## ✨ 项目是什么

`赤血（Red Blood）` 不是单一小说稿，也不是零散灵感堆栈，而是一座持续生长中的修仙 / 玄幻世界观仓库。

这里既有天地底层规则、九州与神族两套文明机器的长期对冲，也有州域、门阀、人物、器物、异类、茶馆闲谈与地方生计。宏大叙事和贴地日常被同时写进同一张互相引用的网里。

这个项目想解决的不是“有没有点子”，而是：

- 让多人可以围绕同一世界持续接力，而不是各写各的平行草稿。
- 让设定具备可检索、可修订、可回链、可追溯的骨架。
- 让 AI 协作真正建立在既有世界观之上，而不是脱离上下文即兴乱编。

## 🪄 为什么适合协作

### 🗂️ 黑曜石（Obsidian）

黑曜石（Obsidian） 在这里不是单纯的笔记软件，而是世界观操作台。

- `[[双链]]` 用来把人物、势力、州郡、器物、历史事件彼此咬合。
- 图谱视图可以直接看见哪些概念已经长成网络，哪些条目仍然孤悬。
- 回链、搜索、大纲视图很适合做设定回填、命名统一和结构整理。

如果需要改文件名，务必在 黑曜石（Obsidian） 里操作。这样它会自动更新双链引用，避免整批链接失效。

### 🌿 Git

Git 在这里承担的是“集体记忆骨架”。

- 每次补设定、重写条目、回填旧文，都会留下清晰的版本脉络。
- 不同作者可以并行推进不同州域、人物或故事线，再通过提交历史回看演化过程。
- 对世界观工程来说，版本控制不只是备份，更是防止设定漂移的重要工具。

### 🤖 协作智能体

这个仓库把 AI 放在“检索、对齐、补缀、扩写”的位置，而不是让它脱离设定直接代写。

- [`AGENTS.md`](./AGENTS.md) 约束检索顺序、交叉引用规范、人物双文件规则与文风边界。
- [`MEMORY.md`](./MEMORY.md) 索引长期有效的协作记忆，帮助后续轮次更快接住上下文。

这样做的目标不是省掉创作判断，而是把多人协作和多轮 AI 协作尽量压进同一套轨道。

## 🚧 现在的进展

如果把这个项目看成一座还在扩建的仙侠城池，那它现在已经过了“先立牌坊”的阶段，正在往“州域能走、人物能活、器物能用、故事能接”这一步稳稳推进。

目前比较明显的进展有：

- 世界底层规则已经立住，而且开始往更高层的“天势”与“回流”推进。`世界背景`、`众生网`、`州城法网`、`修士境界` 与 `回流天势` 已经能互相支撑，不再只是并排摆放的总纲。
- 九州地理骨架已经基本铺开。十三州与外藩不只具名，许多州已经有州志、州城、门阀、地方机构与对应的生活气口，州与州之间的通路感也更清楚了。
- 茶馆线已经从“好读的入口”长成了能持续供血的故事接口。`偏街旧茶馆`、`回青医馆`、`保和丹铺`、时间线、灯下短论与一组茶馆人物，如今已经能反过来承接更大的州城风声与暗线。
- 圣皇序列已经明显成形。`圣皇本纪`、`圣皇纪元编年` 与 `1.中极皇` 到 `13.章皇` 这组条目，让九州这条历史中轴终于有了更稳的时间骨架。
- 九州深水区正在长出真正的阴面厚度。`地府`、`鬼帝`、`承天祖岳`、`坠炎渊`、`昆仑仙池` 这一批条目，让禁地与异类不再只是“危险地点/怪物名单”，而开始承担历史余波、帝统暗影与天地裂口的意义。
- 神族这一侧也从总纲走向了制度细部。`骨舟`、`镜律`、`命册`、`承序白丸` 以及一组神族机构条目，已经把白谕、定序、净化、收割等秩序如何真正落地写得更清楚。
- 经济体系已经补出一套清楚的基础框架。`货币-玄金`、`能源-灵石`、`购买力对照` 与散修生计这一批条目，已经把“什么是钱、什么是能源、不同阶层大致能买到什么”明确下来。
- 这意味着仓库里的日常秩序不再只靠气氛支撑。凡人生计、修士消耗、店铺收入、价格高低与跨州流通，如今都开始有可参照、可落地、也更方便继续扩写的基准线。

换句话说，当前仓库最可贵的地方，不是“条目已经很多”，而是不同层级的内容开始真正互相咬合了。现在继续往前推，重点已经不是单纯扩容，而是让州郡、人物、历史、器物和故事线之间长出更强的承接关系。

## 🤖 `AGENTS.md` 如何接入不同工具

仓库的主规则文件只有一份：根目录 [`AGENTS.md`](./AGENTS.md)。

### Codex

Codex 在仓库根目录启动时，会直接读取当前项目中的 `AGENTS.md`。

### Claude Code

[`.claude/CLAUDE.md`](./.claude/CLAUDE.md) 当前内容为：

```md
@import AGENTS.md
```

这意味着 Claude Code 会通过导入方式复用根目录主规则。

### Gemini CLI

[`.gemini/settings.json`](./.gemini/settings.json) 当前配置为：

```json
{
  "context": {
    "fileName": ["AGENTS.md"]
  }
}
```

这意味着 Gemini CLI 会把根目录 `AGENTS.md` 注入会话上下文。

最稳妥的约定始终是：

- 规则只维护一份主文件：`AGENTS.md`
- Claude Code 通过 `@import` 接入
- Gemini CLI 通过 `context.fileName` 接入
- Codex 在仓库根目录直接读取
