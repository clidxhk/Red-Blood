# 赤血（Red Blood） 🔥

> 一个把仙侠世界观写成可协作知识网络的尝试。  
> 用 Obsidian 串联设定，用 Git 保留演化脉络，用氛围编程工具辅助检索、补缀、扩写与回填。  
> 一起把一座能生长、能回链、能接力的仙侠世界慢慢写出来。 ✨

---

## ✨ 项目是什么

`赤血（Red Blood）` 是一座面向集体创作的世界观仓库。

它不是单一小说稿，也不是零散灵感堆栈，而是一个持续生长中的修仙 / 玄幻设定工程：  
上层有神族、九州、纪元与秩序之争，下层有州域、宗门、人物、器物、茶馆闲谈与地方生计；宏大叙事和贴地日常被同时写进一张互相引用的网里。

这个项目想做的事很直接：

- 🤝 让多人可以围绕同一世界持续接力，而不是各写各的平行草稿。
- 🧭 让设定不只“有点子”，还拥有可以追溯、可修订、可交叉引用的骨架。
- 🛠️ 让氛围编程工具参与创作流程，但不替代世界观判断、风格控制与设定自洽。

---

## 🪄 为什么这样协作

这是一种“写作者 + 知识库 + 版本控制 + AI 辅助”的联合实验。

### 🗂️ 黑曜石（Obsidian）

Obsidian 在这里不是单纯的笔记软件，而是世界观操作台。

- `[[双链]]` 用来把人物、势力、州域、器物、历史事件彼此咬合。
- 图谱视图可以直接看到哪些概念已经长成网络，哪些条目仍然孤悬。
- 回链、搜索、模板和大纲视图很适合做设定回填与结构整理。

### 🌿 Git

Git 在这里承担的是“集体记忆骨架”。

- 每次补设定、改总纲、重写条目，都会留下清晰的版本脉络。
- 不同作者可以并行推进不同州域、人物或场景，再通过提交历史回看演化过程。
- 对世界观项目来说，版本控制不只是备份，更是防止设定漂移的重要工具。

### 🤖 氛围编程工具

氛围编程工具在这个项目中的角色，不是“自动写完”，而是“加速联想、检索、对齐与成稿”。

- 帮助快速扫描仓库里已有设定，避免撞车和断链。
- 在既有文风、命名和关系网基础上做扩写，而不是脱离上下文即兴乱编。
- 适合承担初稿生成、旧文补缀、关联回填、结构整理这类高频工作。

而这个项目之所以适合多人持续协作，不只是因为用了 AI 工具，更因为它给 AI 协作补上了两层很重要的“轨道”：

- [`AGENTS.md`](./AGENTS.md)：负责定义创作规则、检索流程、交叉引用习惯、人物条目规范与文风边界。它像整个项目的协作宪章，能让不同人、不同轮次、不同智能体产出的内容尽量落在同一套世界逻辑里。
- [`MEMORY.md`](./MEMORY.md)：负责索引长期有效的协作记忆，例如用户偏好、项目阶段、写作反馈和参考入口。它不是设定正文本身，而是为了让后续协作者或智能体更快接住上下文，少走弯路、少重复磨合。

两者合起来，等于把“怎么写”“写到什么边界”“需要记住什么长期信息”都提前明确下来。这样一来，氛围编程就不只是灵感加速器，也能成为更规范、更容易上手的协同创作工具。

理想状态不是“让 AI 代写世界”，而是“让更多人更顺畅地一起造世界”。

## 🌌 世界大纲

这个世界目前已经形成了比较清楚的三层骨架。

### 1. 天地母题 🌫️

世界的核心矛盾，不只是修士与凡人的高低，而是谁有资格决定灵气、资源、名分与秩序的流向。

- `背景/世界背景.md` 把天地的底层逻辑落在“众生劳作托举高位秩序”上。
- `背景/众生网.md`、`背景/修士境界.md` 继续把修行、制度、地脉和承接关系织成同一套解释。
- 赤血一脉并不只是反抗口号，而是从底层回路出发，试图重写世界分配逻辑的新火。

### 2. 双重文明对峙 ⚔️

当前世界最鲜明的宏观结构，是九州与神族两套文明机器的长期对冲。

- 九州侧更贴地，重州域、门阀、军镇、书院、宗门、药路、商路与地方承接。
- 神族侧更高位，重定序、血统、官署、州域用途化与温语统治。
- 两边并不是简单善恶对立，而是两种不同层级、不同效率、不同压迫方式的秩序系统。

### 3. 从总纲到局部的落地 🧱

项目已经不止停留在“有一个大世界观”的层次，而是开始往可写故事、可写人物、可写州域生活的层面沉下去。

- 有总纲：九州、神族、世界背景都已有统领性文件。
- 有地理：九州之地总录、各州州志与附属区域正在展开。
- 有势力：门阀、九州巨头、神族机关与地方宗门已开始成型。
- 有角色：九州人物、神族人物和茶馆小人物都在补全。
- 有器物与功法：能支撑更细的制度、战争、日常与修行描写。
- 有场景：茶馆闲聊、历史报告、选集等内容，说明世界已经能承接具体叙事。

## 🚪 推荐打开方式

### 1. 用 Obsidian 打开仓库 🧭

这是最自然的阅读方式。双链、回链、图谱和大纲会让整个项目一下子“活起来”。

### 2. 从总纲进入 📖

推荐先读这些文件：

- `背景/世界背景.md`
- `势力/九州/九州势力总纲.md`
- `势力/神族/神族势力总纲.md`
- `势力/九州/九州之地/九州之地总录.md`
- `大纲/赤龙大纲.md`

### 3. 再往局部深潜 🔎

可以顺着自己最感兴趣的方向往下走：

- 想看宏观秩序：读九州、神族、世界背景
- 想看地方风貌：读各州州志与参考稿
- 想看人物和烟火气：读茶馆闲聊、小人物、地方宗门
- 想看制度与修行：读功法、器物、州城法网、众生网

## 🚀 这个项目想去哪里

理想中的 `Red Blood`，不是一次性完成品，而是一套可以长期演化的共同宇宙底座：

- 可以承载长篇主线，也能容纳支线、短篇、人物外传和地方志。
- 可以被不同作者接力扩写，但仍维持统一命名、统一逻辑和统一气质。
- 可以把“氛围感”“史诗感”“烟火气”和“制度感”同时写进一个修仙世界里。

如果说很多世界观项目停在“好看”，那这个项目更想继续往前一步，做到“可协作、可积累、可生长”。
