---
name: character-relationship-interactions
description: 新增人物时要先补清与既有人物的关系、身份秩序和互动方式，不能只立单人设定
type: feedback
---

规则：新增或扩写人物时，必须先把他与现有角色的关系线补清，再把这种关系落到具体互动里。若人物与既有角色存在主从、租户与东家、旧识、债主债户、同门、上下级或长期熟客关系，正文里就要写出称呼、站位、说话分寸、替谁圆话、敢不敢顶嘴等具体表现，不能只在人物卡里抽象写“关系密切”或“互相敬防”。

**原因：** 用户明确指出，增加人物时必须考虑现有人物之间的关系与互动，比如陆照麟是程衡生的东家，那么程衡生在场时就应体现出捧着东家、替东家说好话、但又不至于失真过头的社会性互动。

**应用场景：** 新建茶馆小人物、补人物卡、给群像增配角、回填旧文匿名角色、编写同场对话和势力内部互动时，先核实现有关系，再把身份秩序和互动动作写进正文与 json 的 relations / hooks。
