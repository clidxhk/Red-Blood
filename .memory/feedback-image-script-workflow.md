---
name: Image Script Workflow
description: 生成并插入各类图片时统一复用固定脚本流程，不再重复摸索取参、审图和落图步骤
type: feedback
---

生成并插入各类图片时，统一使用固定脚本流程：先检索目标条目与同类条目的现有插图位置、画面类型和内容约束；再读取 `scripts/generate_image_from_md.py` 的当前行为，确认脚本会自动从 `scripts/.env` 读取 API key、base URL 和 image model；随后直接运行脚本，只传 Markdown 路径、额外画面要求、尺寸、质量和输出目录，不再额外排查环境变量或手动传 key、model、base URL；生成后检查图片是否贴合条目需求，再把首张输出改名为与条目同名的 `png`，并按该类文档的既有版式插回正文合适位置。若画面不准，优先重写 `--extra` 的画面描述再重跑，不重复走环境排查。

**原因：** 用户希望把脚本使用方法固化下来，之后处理任何图片任务时都不必每次重新检索一遍脚本取参、审图和插图流程。

**应用场景：** 当用户要求为人物、器物、势力、州域、禁地、场景等任意条目用脚本生成图片，并插回对应 Markdown 正文时。
