---
name: 人物条目必须补json
description: 常规人物设定需同步产出 Markdown 正文与 JSON 底谱，但异类条目默认不构建人物底谱。
type: feedback
---

规则：凡涉及常规人物条目的新建、扩写或改写，必须同时交付 `md` 正文与 `json` 底谱，不得只写人物正文；但“异类”条目默认只写正文，不再同步构建人物 `json` 底谱，除非用户另行要求。

**原因：** 用户先前要求人物设定必须补 `json`，本轮进一步明确异类不按常规人物处理，不需要构建人物底谱，以免异类条目被人物结构约束得过重。

**应用场景：** 后续新增、补完、重写常规人物时继续同步维护对应 `json`；若任务对象属于异类、灵兽、器灵、妖魔等异类条目，则默认只写 `md` 正文，并避免额外生成人物底谱。
