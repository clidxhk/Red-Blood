name: teahouse-reference-boundary
description: 茶馆闲聊默认不直接把历史报告、东阳选集等读者向或他场景向材料塞进茶馆众人的口中，只作幕后参考
type: feedback
---

规则：以后写茶馆闲聊时，像[[历史报告-帝阙的成型]]、东阳选集这类主要写给读者看的总纲、史论、论说或其他场景向文本，默认只可作为幕后参考依据，不直接放进茶馆众人的参考框架、口头引用或自然议论里。

规则：茶馆众人的说法应立足他们各自能接触到的街面风声、亲见经历、行当消息、旧闻残片、地方传说与身份所能承受的认知边界，不可把明显超出他们日常信息来源的宏观文本直接端进馆里。

规则：若用户明确要求必须写入某篇茶馆正文，才可直接引用这类材料；即便写入，也应先做场景内转化，不能生硬像作者注释或背景资料搬运。

**原因：** 用户明确要求，[[历史报告-帝阙的成型]]、东阳选集等材料本质上更偏向读者说明层或其他场景层，不能简单直接拿来放进茶馆众人的参考中，否则会破坏茶馆人物的认知边界与场景真实感。

**原因：** 用户进一步说明，这类文本可以作为创作时的参考资料，但默认不能直接引用，除非任务明确要求必须写进去。

**应用场景：** 续写、补写、重写任何“场景/茶馆闲聊”条目时，先区分幕后依据与角色可说信息。历史报告、东阳选集、总纲式设定、论说体正文可用于作者侧检索和设定抽取，但落到茶馆里时，要改写成街谈、见闻、旧人回忆、地方传闻或角色能接触到的二手消息；若用户特别指定要保留某条直接引用，再单独处理。
