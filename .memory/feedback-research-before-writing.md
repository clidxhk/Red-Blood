name: research-before-writing-preference
description: 用户要求每次写设定前先检索仓库内相关内容，以现有依据为基础扩写，避免脱离上下文即兴编写
type: feedback
---

规则：用户要求撰写设定、条目或世界观内容时，必须先检索仓库内是否已有相关条目、提及或上下文依据，再基于现有内容扩写；不得未经查证就直接即兴编写。涉及时间线、势力入场顺序、来源归属、地理范围时，必须额外核对关键事实，不能把别处常见设定或同类元素直接套进当前世界观。若执行不稳定，应直接强化 Agent 提示词，把“先检索、再提取、后写作”写成硬性步骤与禁止项，而不是依赖模型自行领会。

**原因：** 用户明确指出“我让你写什么，你应该首先检索有没有相关内容，而不应该自己马上开始瞎编，必须每次写作都要有根据”，并进一步反馈“还是有问题约束还是不够，直接改Agent使约束更强一些”；在重写神族相关条目时，又补充强调“神族是章皇末年才外来的，不能忘了这类关键前提直接瞎写”。

**应用场景：** 新增或扩写人物、势力、地点、传说、体系、器物等一切创作型内容前，先做仓库检索与上下文阅读，再决定写法、落点与交叉引用；若发现代理经常跳过检索，优先修改代理规则本身。
